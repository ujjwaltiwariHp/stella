# Review Rubric

Score the submission on:

1. Send speed: Does it return quickly after provider acceptance while preserving safety, duplicate protection, and audit/bookkeeping work?
2. Practicality: Is the code small, understandable, and easy to move into Stella?
3. Monthly automation: Does it select tenants using balance, DPD, cadence history, holds, and recent communications?
4. Communication: Did they explain assumptions and tradeoffs without overcomplicating the task?
5. Production awareness: Did they keep everything local/offline and avoid credentials or live-provider assumptions?

Strong candidates should also point out any data-quality issues they noticed in the fixtures and how they would verify this in a real Stella branch.
